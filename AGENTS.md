# Changelog

Substantive changes to this repo are logged in `CHANGELOG.md`.
When implementing a plan or making any non-trivial edit, add a dated
entry there. Keep entries concise — a couple of git-commit-style bullet
points, not a blow-by-blow.
- Omit formatting-only, whitespace, or other mechanical changes from the changelog.
- Located at `CHANGELOG.md` in the project root.
- Covers substantive changes across the **entire repo**, not a single subsystem.
- Headers are newest-first (reverse chronological). Within a date, bullets are
  sorted alphabetically by `**tag**`.
- Single header per date (merge any duplicate date sections).
- No blank lines between bullets in a section; one blank line between sections.
